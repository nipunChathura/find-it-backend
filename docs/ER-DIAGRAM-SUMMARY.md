# Find It – ER Diagram Summary

The **Find It Entity Relationship Diagram** describes the database schema for the backend. It includes **users** (linked by role to merchant, sub-merchant, or customer via optional foreign keys), **customers**, **merchants** and **sub_merchants** (one-to-many; both can own outlets), **outlets** (with location, bank, subscription, and onboarding fields, and references to **provinces**, **districts**, and **cities** in a hierarchy), **categories** and **items** (items belong to a category and an outlet), **discounts** and **discount_items** (many-to-many link between discounts and items), **customer_favorites** (customer–outlet with optional nickname), **customer_search_history**, **notifications**, **payments** and **outlet_schedule** (per outlet), **holiday_master**, and **feedbacks** (customer rating and comment for outlets). Relationships are documented with cardinalities (e.g. merchant/sub-merchant to outlets, province–district–city, outlet to items/payments/schedules/feedbacks/favorites). The diagram is defined in Mermaid in the docs and a PNG overview is available; most tables use an abstract entity for audit fields (created_by, modified_by, version), which are omitted from the diagram for clarity.
