# Find It – System Architecture Summary

The Find It system follows a three-tier architecture. **Clients**—Admin and Merchant web applications and the Customer web or mobile app—communicate with a **single backend** over HTTPS using REST APIs and JWT for authentication. The **Find It backend** is a Spring Boot application (context path `/find-it`, default port 9090) that exposes a REST API, enforces security (JWT, BCrypt, CORS, role-based access), and implements business logic and data access via JPA. All persistent data is stored in a **MySQL** database; uploaded files (e.g. profile, item, receipt, and discount images) are stored in a **local file system** directory (configurable, e.g. `./images`). The backend is stateless (no server-side sessions), so it can be scaled horizontally. In short: one backend serves all client types, uses MySQL for data and a local folder for images, and clients access it via HTTPS with JWT.
